Manuel utilisateur de OctopusViewer

OctopusViewer rassemble tous vos médias Omeka S dans une visionneuse légère. Elle est prévue pour être utilisée en remplacement d’autres visionneuses comme UniversalViewer ou Mirador.

Motivation

UniversalViewer et Mirador sont d’excellents outils mais ils ont quelques limites quand ils sont utilisés dans Omeka S:

  • Ils nécessitent un manifeste IIIF, ce qui veut souvent dire que des modules supplémentaires doivent être installés

  • Ils supportent seulement un sous-ensemble des types de médias Omeka S

  • Ils ne sont pas faciles à personnaliser dans les thèmes Omeka S

Les objectifs de OctopusViewer sont:

  • il doit être capable d’afficher tous les types de média, avec un affichage par défaut sensé dans les cas où le média ne peut pas être affiché dans un navigateur

  • il doit utiliser le moins de dépendances externes possible, en privilégiant les outils déjà disponibles dans Omeka S (comme OpenSeadragon)

  • il ne doit dépendre d’aucun autre module Omeka S, mais doit pouvoir être étendu par d’autres modules

  • il doit être facilement personnalisable dans les thèmes Omeka S

Fonctionnalités

  • Une visionneuse divisée en 3 panneaux:

    • le panneau de gauche contient un sélecteur de média

    • le panneau central affiche le média

    • le panneau de droite affiche les métadonnées du média sélectionné

  • Les images sont affichées dans OpenSeadragon, ce qui permet de zoomer, faire pivoter, et retourner les images

  • Un serveur d’image IIIF optionnel peut-être utilisé

  • Les fichiers PDF sont affichés dans le lecteur PDF intégré au navigateur, s’il existe. Si le navigateur ne supporte pas les PDF (comme la plupart des navigateurs sur mobile), pdfjs est utilisé.

  • Mode plein écran
